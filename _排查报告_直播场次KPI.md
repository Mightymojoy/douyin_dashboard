# 抖音店铺看板 · 直播场次KPI数据排查报告

## 一、排查范围
- **筛选条件**：全部店铺，2026-06-01 至 2026-07-07，直播场次 Tab
- **数据源**：`E:\抖音店铺-看板\抖音自营店铺数据源.xlsx`（三店 Sheet 逐行汇总核对）
- **问题**：红圈 KPI（比率、效率、客单价、人均观看等）显示值与数据源汇总计算不一致

## 二、根因定位
**Dashboard 代码错误地把「每日已算好的比率/效率值」做简单算术平均，而不是按整个周期重新汇总计算。**

错误示例：
```js
// 错误：把每天的曝光-观看率加起来 ÷ 天数
expViewRate: d.reduce((s,r)=>s+(r.expViewRate||0),0)/d.length*100
```

正确做法：
```js
// 正确：整个周期 UV 总和 ÷ 直播曝光总和
expViewRate: d.reduce((s,r)=>s+(r.uv||0),0) / d.reduce((s,r)=>s+(r.liveExpose||0),0) * 100
```

受影响的指标：所有「率」「效率」「均价」「人均观看」类，具体见下表。

## 四、修复内容
- 修改文件：`douyin_dashboard.html`
- 修复范围：直播场次 Tab 的 `SS_KPI_META` 中所有比率、效率、均价、退款率、人均观看的 `calc` 函数。
- 修复原则：
  - **率类**：分子分母分别汇总，再相除；
  - **效率类**：总量 ÷ 总时长；
  - **均价**：总金额 ÷ 总人数；
  - **人均观看**：按 UV 加权平均；
  - **退款率**：退款金额 ÷ GMV。
